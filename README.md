# NxN Systolic Array

A systolic array is a specialized hardware accelerator for matrix-matrix multiplication. My implementation arranges N×N processing elements (PEs) in a grid, each performing fixed-point 16-bit multiply-accumulate operations. What makes it systolic is the way inputs and outputs propagate through the array, ensuring that data arrives at each PE at precisely the right time for multiplication and results flow out from each PE to be accumulated. The finite state machine controls instruction decoding, data feeding, matrix tiling/chaining, and result draining. This implementation is based on Coding Test 4 in the [specification](eic-coding-tests.pdf) from the Efficient and Intelligent Computing Lab at Georgia Tech.

## Requirements

### Option 1: Nix Flake (Recommended)

If you have Nix installed, the flake handles all dependencies:

```bash
nix develop
# now all dependencies are installed
cd src
make sim
exit  # or Ctrl+D to leave the nix environment
```

### Option 2: Manual Install

- **Python 3.11+** with `numpy` and `matplotlib`
- **Icarus Verilog 12.0+**
- **Yosys 0.55+**
- **GNU Make**

## Usage

All commands are run from the `src/` directory. The Makefile supports several modes.

### Flags
`N` is an integer (`N=4`) or a list of integers (`N=4,8`)

`I` is a comma-separated list of zero-terminated integers (`I=32,16,64,8,16,0`)

`seed` can be any integer and is used when randomizing the input matrices

`wdir` can be used to select a different working directory (`make clean` won't clear custom working directories)

### Clean
Removes all generated simulation artifacts in `working_dir/`
```bash
make clean
```

### Identity tests
`gen_inputs.py` fills the weight matrix with an identity matrix

```bash
make sim N=4
```
Executes a 4x4 identity multiply on a 4x4 array to verify M(4x4) x I(4x4) = M(4x4). Verifies correctness on a simple matrix multiply the same size as the array.
```bash
make sim N=4 I=8,0
```
Executes a 8x8 identity multiply on a 4x4 array to verify M(8x8) x I(8x8) = M(8x8). Verifies correctness when tiling for square matrixes.

```bash
make sim N=4 I=8,4,0
```
Executes a 8x4 identity multiply on a 4x4 array to verify M(8x4) x I(4x4) = M(8x4). Verifies correctness when tiling for non-square matrixes.

### Actual simulations
`gen_inputs.py` fills the weight matrix with a random matrix


```bash
make sim N=4,8 I=32,16,64,8,16,0
```
Executes (32x16) x (16x64) x (64x8) x (8x16) = (8x16) on 4x4 and 8x8 arrays. Verifies correctness of custom instruction sequences against the reference output generated by numpy.

### Memory Layout

#### Memory A (Input Matrix)
- **Type**: Single-port SRAM with asynchronous read and synchronous write
- **Width**: 16-bit (signed Q8.8 fixed-point)
- **Format**: Row-major flattened matrix
- **Access**: Read during compute; written during chaining to store intermediate result as input for next MMM
- **Files**: `data_mem.v` (initialized with `mem_a.hex`)

For MMM 0, Memory A holds the input matrix. For subsequent MMMs in a chain, the previous output is converted back to Q8.8 and written into Memory A by the state machine.

#### Memory B (Identity/Weight Matrices)
- **Type**: Single-port SRAM with asynchronous read and synchronous write
- **Width**: 16-bit (signed Q8.8 fixed-point)
- **Format**: Row-major flattened, concatenated weight matrices for all MMMs in the chain
- **Access**: Read-only during compute; weight matrices for each MMM are stored back-to-back
- **Files**: `data_mem.v` (initialized with `mem_b.hex`)

The state machine maintains a base address pointer (`b_base`) and advances it after each MMM to fetch the next weight matrix.

#### Memory I (Instruction Sequence)
- **Type**: Single-port SRAM with asynchronous read and synchronous write
- **Width**: 32-bit unsigned
- **Format**: Sequence of matrix dimensions: `[A, B, C, ..., 0]`
  - Each adjacent triplet `(A, B, C)` specifies one matrix multiply: (AxB) x (BxC) = (AxC)
  - `0` terminates the sequence
- **Access**: Read-only during compute
- **Files**: `instr_mem.v` (initialized with `mem_i.hex`)

#### Memory O (Output Data)
- **Type**: Dual-port SRAM (one read port, one write port)
- **Width**: 32-bit (signed Q16.16 fixed-point accumulator)
- **Format**: Row-major flattened matrix
- **Access**: Written after draining; read during chaining to get intermediate result as input for next MMM
- **Files**: `output_mem.v` (final output is in `sim_n<N>/sim_out_n<N>.hex`)

### Array Layout

#### Processing Element (`systolic_pe.v`)

Each PE is a single-cycle MAC unit:

```
                      in_b (from above PE)
                        ↓
in_a (from left PE) ─→ [x] ─→ out_a (to right PE)
                       acc
                        ↓
                      out_b (to below PE)

MAC Operation: acc_next = acc + (in_a * in_b)
```

**Signals:**
- `clear`: Reset accumulator to 0
- `compute`: Enable MAC operation
- `drain`: Enable systolic drain (pass accumulator downstream)
- `in_a`: 16-bit signed input from left
- `in_b`: 16-bit signed input from above
- `out_a`: 16-bit signed output to right PE (one-cycle pipelined)
- `out_b`: 16-bit signed output to below PE (one-cycle pipelined)
- `drain_in`: 32-bit output accumulator from above PE (for systolic shift)
- `drain_out`: This PE's accumulator sent downstream
- `acc`: Internal 32-bit accumulator (read by state machine for result collection)

#### Systolic Array (`systolic_array.v`)

NxN grid of PEs with systolic data flow:

```
           B[0][0]      B[0][1]      B[0][2]     ...      B[0][N-1]
              ↓            ↓            ↓         ↓           ↓
 A[0]  →  PE[0][0]  →  PE[0][1]  →  PE[0][2]  →  ...  →  PE[0][N-1]
              ↓            ↓            ↓         ↓           ↓
 A[1]  →  PE[1][0]  →  PE[1][1]  →  PE[1][2]  →  ...  →  PE[1][N-1]
              ↓            ↓            ↓         ↓           ↓
 A[2]  →  PE[2][0]  →  PE[2][1]  →  PE[2][2]  →  ...  →  PE[2][N-1]
              ↓            ↓            ↓         ↓           ↓
 ...   →     ...    →     ...    →     ...    →  ...  →      ...
              ↓            ↓            ↓         ↓           ↓
A[N-1] → PE[N-1][0] → PE[N-1][1] → PE[N-1][2] →  ...  → PE[N-1][N-1]
              ↓            ↓            ↓         ↓           ↓

Each PE performs a single MAC operation and passes its result down
```

**Key Features:**
- Input A shifts **left-to-right** along rows; row `i` is delayed by `i` cycles
- Input B shifts **top-to-bottom** along columns; column `j` is delayed by `j` cycles
- Ensures A[i][k] and B[k][j] meet at PE[i][j] simultaneously
- PE accumulators chain vertically for systolic drain (bottom row outputs in order)

**Signals:**
- `clear`, `compute`, `drain`: Control signals broadcast to all PEs
- `a_in_flat`: Nx16-bit packed input vector (one element per row)
- `b_in_flat`: Nx16-bit packed input vector (one element per column)
- `acc_flat`: NxNx32-bit packed accumulator readout (debug)
- `drain_flat`: Nx32-bit packed drain output (results shift out bottom row)

---

### State Machine (`state_machine.v`)

The FSM orchestrates instruction decoding, matrix loading, systolic computation, result draining, and output writeback. It supports tiling for matrices larger than N and chaining for multiple MMMs.

#### States

| State       | Purpose                                                                 |
|-------------|-------------------------------------------------------------------------|
| `IDLE`      | Wait for `ap_start` pulse to begin                                      |
| `FETCH_I`   | Set instruction address to current instruction pointer (`ip`)           |
| `FETCH_A`   | Read/latch `dim_a`; detect end-of-program (`idata == 0`)                |
| `FETCH_B`   | Read/latch `dim_b`; advance to setup for current MMM                    |
| `SETUP`     | Initialize tiling tile counters and buffer pointers                     |
| `SETUP_WAIT`| Wait for one cycle (housekeeping)                                       |
| `FEED_A1`   | Stream first tile of A (before compute starts)                          |
| `FEED_A2`   | Continue streaming A as compute progresses                              |
| `FEED_B1`   | Stream first tile of B (before compute starts)                          |
| `FEED_B2`   | Continue streaming B as compute progresses                              |
| `PUSH`      | Finish pushing A and B into array (final skew cycles)                   |
| `FLUSH`     | Wait for partial products to accumulate inside array                    |
| `FLUSH_WAIT`| Extra cycle before drain                                                |
| `DRAIN`     | Shift PE accumulators down by one row (systolic drain step)             |
| `DRAIN_WAIT`| Wait between drain cycles                                               |
| `DRAIN_READ`| Write current drained bottom-row values into output memory              |
| `NEXT`      | Advance tile indices; after final tile, move to `CHAIN_I`               |
| `CHAIN_I`   | Read next instruction triplet for chained MMM                           |
| `CHAIN_D`   | Prepare chained state (convert output Q16.16 back to Q8.8 for input)    |
| `WB1`       | Writeback converted output into memory A for next MMM                   |
| `WB2`       | Finalize writeback; continue `WB1` or restart at `SETUP_WAIT`           |
| `DONE`      | Assert `ap_done` and wait for next `ap_start`                           |

#### Transitions
```
IDLE → FETCH_I → FETCH_A → FETCH_B → SETUP → SETUP_WAIT
  → FEED_A1 → FEED_A2 → FEED_B1 → FEED_B2 → PUSH
  → FLUSH → FLUSH_WAIT → DRAIN → DRAIN_WAIT → DRAIN_READ
  → NEXT (MORE TILES? YES → SETUP_WAIT → ... : NO → CHAIN_I → ...)
  → CHAIN_I → CHAIN_D (MORE MMMS? YES → WB1 → WB2 → SETUP_WAIT → ... : NO → DONE)
  → DONE
```
